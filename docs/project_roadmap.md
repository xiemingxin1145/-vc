# 三国人生模拟器项目整理清单

## 当前项目状态

仓库已经具备 Android / Kotlin / Jetpack Compose 基础结构，并包含角色创建、年度推进、属性成长、随机事件、战斗、城池、市场、关系、子女、存档和生涯记录等核心模块。

最近新增内容：

- `SanguoContentPack.kt`：高级三国剧情事件包。
- `SanguoGallery.kt`：事件插图库索引。
- `SanguoAchievementEngine.kt`：成就判断引擎。
- `res/drawable/illust_*.xml`：六张三国事件矢量插图。

## 当前最大问题

项目不是缺框架，而是缺整理和接入：

1. 新增剧情包尚未完全接入年度事件池。
2. 插图资源尚未显示到事件弹窗中。
3. 成就引擎尚未接入主界面。
4. 缺少稳定构建验证。
5. 事件内容量还不够，重复游玩时容易枯燥。
6. UI 反馈感还不够强，升级、胜利、成就、结局缺少仪式感。

## 第一阶段：稳定接入

目标：不大改架构，把已加入的新模块真正显示出来。

任务：

- 在 `EventChoiceDialog` 中显示事件插图。
- 根据 `event.id` 从 `SanguoContentPack` 或 `SanguoGallery` 找到对应图片。
- 在主界面增加“成就”入口，展示 `SanguoAchievementEngine` 判断结果。
- 在主界面或标题页增加“图库”入口，展示六张事件插图。
- 保证不改数据库结构，避免破坏存档。

完成后效果：

- 年度事件不再只是文字弹窗，而是图文事件。
- 玩家能看到成就达成状态。
- 项目内容开始形成系统感。

## 第二阶段：扩充事件池

目标：让游戏真正耐玩。

建议事件规模：

- 普通生活事件：30 条
- 历史大事件：20 条
- 官场事件：20 条
- 经商事件：20 条
- 战争事件：20 条
- 名人互动事件：30 条
- 家族子女事件：20 条

第一版目标总数：100 条以上。

接入方式：

- 先继续使用 `RandomEvent` 数据结构。
- 不急着新增复杂事件类型。
- 每个事件至少包含：标题、描述、三选项、检定属性、检定难度。
- 高级事件按年份、势力、属性、声望进行筛选。

## 第三阶段：结局和列传

目标：让玩家一局结束后有爽感。

任务：

- 强化 `GameOverScreen`。
- 根据属性、势力、城池、金钱、名望、年龄生成称号。
- 显示一段列传式总结。
- 加入结局等级：SS、S、A、B、C。
- 展示代表插图或印章。

结局示例：

- 乱世枭雄
- 汉室忠臣
- 富甲天下
- 一代军神
- 青史谋主
- 归隐山林

## 第四阶段：APK 构建验证

目标：保证项目可以稳定打包。

建议运行：

```bash
./gradlew assembleDebug
```

如有错误，优先修：

1. Kotlin 编译错误
2. drawable 资源错误
3. Compose import 缺失
4. Room / KSP 配置问题
5. Android SDK 版本问题

## 第五阶段：美术升级

当前插图是轻量矢量占位图，优点是安全、小、稳定，缺点是冲击力一般。

后续可替换为正式插图：

- 黄巾起义
- 洛阳夜行
- 隆中对策
- 赤壁东风
- 丝路商队
- 宫墙暗流
- 武将单挑

- 结局列传
